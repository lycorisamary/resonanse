# Архитектура проекта Резонанс

## Общая архитектура

Проект разделен на два основных компонента:
- **Backend** (Python/FastAPI) - REST API сервер
- **Frontend** (Flutter/Dart) - мобильное приложение

### Принципы проектирования

1. **Разделение ответственности**: Каждый модуль отвечает за свою область
2. **Типизация**: Все данные типизированы (Type Hints в Python)
3. **Асинхронность**: Использование async/await для работы с БД
4. **Безопасность**: Пароли хешируются, JWT для аутентификации
5. **Валидация**: Pydantic схемы для валидации входящих данных

## Архитектура базы данных

### Текущие таблицы

#### users
Основная таблица пользователей.

**Поля:**
- `id` (Integer, PK) - уникальный идентификатор
- `email` (String, Unique) - email пользователя
- `hashed_password` (String) - хешированный пароль (bcrypt)
- `first_name` (String, Nullable) - имя
- `last_name` (String, Nullable) - фамилия
- `bio` (Text, Nullable) - описание профиля
- `avatar_url` (String, Nullable) - URL аватара
- `is_active` (Boolean) - активен ли пользователь
- `is_verified` (Boolean) - верифицирован ли email
- `latitude` (String, Nullable) - широта
- `longitude` (String, Nullable) - долгота
- `created_at` (DateTime) - дата создания
- `updated_at` (DateTime) - дата обновления

### Будущие таблицы

#### swipes
Таблица свайпов (будущая реализация).

**Особенность**: Одна строка на пару пользователей для избежания гонки потоков.

**Поля (планируемые):**
- `id` (Integer, PK)
- `swiper_id` (Integer, FK -> users.id) - кто свайпнул
- `swiped_id` (Integer, FK -> users.id) - на кого свайпнули
- `direction` (Enum: 'left', 'right') - направление свайпа
- `created_at` (DateTime)

**Индексы:**
- Уникальный индекс на (swiper_id, swiped_id)

#### matches
Таблица совпадений (будущая реализация).

**Поля (планируемые):**
- `id` (Integer, PK)
- `user1_id` (Integer, FK -> users.id)
- `user2_id` (Integer, FK -> users.id)
- `created_at` (DateTime)

#### interests
Таблица интересов (будущая реализация).

**Поля (планируемые):**
- `id` (Integer, PK)
- `user_id` (Integer, FK -> users.id)
- `interest_name` (String)
- `weight` (Float) - вес интереса (для 3D визуализации)

## Безопасность

### Хеширование паролей

Используется библиотека `passlib` с алгоритмом `bcrypt`.

```python
# Хеширование при регистрации
hashed_password = get_password_hash(plain_password)

# Проверка при входе
is_valid = verify_password(plain_password, hashed_password)
```

### JWT токены

- **Алгоритм**: HS256
- **Время жизни**: Настраивается через `ACCESS_TOKEN_EXPIRE_MINUTES` (по умолчанию 7 дней)
- **Секретный ключ**: Хранится в переменной окружения `SECRET_KEY`

**Структура токена:**
```json
{
  "sub": "user_id",
  "exp": 1234567890
}
```

### CORS

Настроен для работы с Flutter приложением. В разработке разрешены все источники (`*`).

## Коммуникация Frontend-Backend

### REST API

- **Протокол**: HTTP/HTTPS
- **Формат данных**: JSON
- **Аутентификация**: JWT токен в заголовке `Authorization: Bearer <token>`

### Пример запроса

```http
POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password123"
}
```

### Пример ответа

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```
