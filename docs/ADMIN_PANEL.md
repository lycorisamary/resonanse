# Админ-панель

## Обзор

Админ-панель предоставляет полный доступ к управлению пользователями системы. Доступна только для администратора, который определяется по email в настройках.

## Настройка доступа

### 1. Настройка администратора

В файле `backend/.env` укажите email администратора:

```env
ADMIN_EMAIL=admin@resonans.ru
```

**ВАЖНО:** Только пользователь с этим email сможет войти в админ-панель.

### 2. Создание администратора

Создайте пользователя с email, указанным в `ADMIN_EMAIL`:

```bash
# Через Swagger или curl
curl -X POST "http://localhost:8000/api/v1/auth/register" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@resonans.ru",
    "password": "adminpassword123",
    "first_name": "Администратор"
  }'
```

### 3. Вход в админ-панель

1. Войдите в приложение с учетными данными администратора
2. На главном экране появится иконка админ-панели (🛡️)
3. Нажмите на иконку для входа в админ-панель

## Функционал админ-панели

### Просмотр пользователей

- Список всех пользователей с основной информацией
- Поиск по email, имени, фамилии, городу
- Индикаторы статуса (активен/неактивен, администратор)
- Сортировка по дате создания (новые первыми)

### Управление пользователями

**Доступные действия:**

1. **Просмотр деталей** - нажмите на пользователя для просмотра полной информации
2. **Активация/Деактивация** - длительное нажатие на пользователя → выбрать действие
3. **Удаление** - длительное нажатие → удалить (с подтверждением)

### Безопасность

- Доступ проверяется на каждом запросе
- Администратор не может удалить самого себя
- Все действия логируются (в будущем)

## API Эндпоинты

### Получение списка пользователей

**GET** `/api/v1/admin/users`

**Параметры запроса:**
- `skip` (int, optional) - количество пропущенных записей (пагинация)
- `limit` (int, optional) - максимальное количество записей (по умолчанию 100)

**Ответ:**
```json
[
  {
    "id": 1,
    "email": "user@example.com",
    "first_name": "Иван",
    "last_name": "Иванов",
    "city": "Москва",
    "is_active": true,
    "is_admin": false,
    ...
  }
]
```

### Получение пользователя по ID

**GET** `/api/v1/admin/users/{user_id}`

### Обновление пользователя

**PATCH** `/api/v1/admin/users/{user_id}`

**Тело запроса:**
```json
{
  "first_name": "Новое имя",
  "city": "Санкт-Петербург",
  "is_active": true
}
```

### Активация пользователя

**POST** `/api/v1/admin/users/{user_id}/activate`

### Деактивация пользователя

**POST** `/api/v1/admin/users/{user_id}/deactivate`

### Удаление пользователя

**DELETE** `/api/v1/admin/users/{user_id}`

**Ограничения:**
- Нельзя удалить самого себя
- Удаление необратимо

## Использование в Flutter

Админ-панель доступна автоматически для пользователя с email из `ADMIN_EMAIL`:

1. Войдите в приложение
2. На главном экране появится иконка админ-панели
3. Нажмите для входа

## Тестирование через Swagger

1. Войдите через `/auth/login` с email администратора
2. Скопируйте токен
3. Нажмите "Authorize" и вставьте токен
4. Протестируйте эндпоинты `/admin/*`

## Безопасность

- Проверка доступа на каждом запросе
- Email администратора хранится в переменных окружения
- В продакшене рекомендуется использовать более сложную систему ролей
