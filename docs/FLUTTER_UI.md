# Flutter UI - Руководство пользователя

## Обзор

Приложение Резонанс имеет полноценный пользовательский интерфейс для:
- Регистрации новых пользователей
- Входа в систему
- Просмотра и редактирования профиля
- Загрузки аватара

## Структура экранов

### 1. Экран входа (LoginScreen)

**Путь:** `lib/screens/login_screen.dart`

**Функционал:**
- Вход по email и паролю
- Валидация полей
- Переход на регистрацию
- Автоматическое сохранение токена

**Поля:**
- Email (обязательное)
- Пароль (обязательное, минимум 8 символов)

### 2. Экран регистрации (RegisterScreen)

**Путь:** `lib/screens/register_screen.dart`

**Функционал:**
- Регистрация нового пользователя
- Валидация всех полей
- Подтверждение пароля
- Автоматический вход после регистрации

**Поля:**
- Имя (необязательное)
- Фамилия (необязательное)
- Email (обязательное)
- Пароль (обязательное, минимум 8 символов)
- Подтверждение пароля (обязательное)

### 3. Главный экран (HomeScreen)

**Путь:** `lib/screens/home_screen.dart`

**Функционал:**
- Приветственный экран после входа
- Навигация к профилю
- Выход из системы
- Проверка авторизации при запуске

### 4. Экран профиля (ProfileScreen)

**Путь:** `lib/screens/profile_screen.dart`

**Функционал:**
- Просмотр информации о пользователе
- Редактирование профиля
- Загрузка аватара из галереи или камеры
- Обновление данных (имя, фамилия, био, дата рождения)

## Навигация

Приложение использует автоматическую навигацию на основе статуса авторизации:

1. **При запуске:**
   - Проверяется наличие сохраненного токена
   - Если токен есть → HomeScreen
   - Если токена нет → LoginScreen

2. **После входа:**
   - Токен сохраняется локально
   - Переход на HomeScreen

3. **После выхода:**
   - Токен удаляется
   - Переход на LoginScreen

## Использование

### Запуск приложения

```bash
cd frontend
flutter pub get
flutter run
```

### Настройка API URL

Перед запуском настройте URL API в `lib/config/api_config.dart`:

```dart
// Для Android эмулятора
static const String baseUrl = 'http://10.0.2.2:8000';

// Для физического устройства (замените на IP вашего компьютера)
static const String baseUrl = 'http://192.168.1.100:8000';

// Для iOS симулятора
static const String baseUrl = 'http://localhost:8000';
```

### Первый запуск

1. Запустите приложение
2. Нажмите "Нет аккаунта? Зарегистрироваться"
3. Заполните форму регистрации
4. После регистрации вы автоматически войдете в систему
5. Нажмите на иконку профиля для просмотра и редактирования

## Особенности UI

### Валидация форм

- Email проверяется на корректность формата
- Пароль должен быть не менее 8 символов
- При регистрации пароли должны совпадать
- Все обязательные поля помечены звездочкой (*)

### Обработка ошибок

- Ошибки отображаются в виде SnackBar
- Красный цвет для ошибок
- Зеленый цвет для успешных операций
- Индикаторы загрузки во время запросов

### Загрузка аватара

1. Нажмите на иконку камеры на аватаре
2. Выберите источник:
   - Галерея
   - Камера
3. Изображение автоматически загрузится на сервер
4. Аватар обновится в интерфейсе

## Технические детали

### Сохранение токена

Токен сохраняется в `SharedPreferences` с ключом `auth_token`.

### Проверка авторизации

При каждом запуске приложения проверяется наличие токена через `ApiService.isAuthenticated()`.

### Обработка ошибок API

Все ошибки API обрабатываются и отображаются пользователю в понятном виде.

## Устранение неполадок

### Приложение не подключается к API

1. Проверьте, что бэкенд запущен: `http://localhost:8000/health`
2. Проверьте настройку `baseUrl` в `api_config.dart`
3. Для физического устройства убедитесь, что телефон и компьютер в одной сети

### Ошибка "Токен не найден"

1. Выйдите из приложения и войдите снова
2. Проверьте, что регистрация/вход прошли успешно

### Ошибка загрузки аватара

1. Проверьте, что MinIO запущен
2. Проверьте размер файла (максимум 5 МБ)
3. Убедитесь, что формат файла поддерживается (JPG, PNG, WEBP)

### 5. Админ-панель (AdminPanelScreen)

**Путь:** `lib/screens/admin_panel_screen.dart`

**Доступ:** Только для пользователя с email из `ADMIN_EMAIL` в настройках бэкенда.

**Функционал:**
- Просмотр списка всех пользователей
- Поиск пользователей
- Просмотр детальной информации о пользователе
- Активация/деактивация пользователей
- Удаление пользователей

**Навигация:**
- Доступна через иконку админ-панели на главном экране (только для администратора)
